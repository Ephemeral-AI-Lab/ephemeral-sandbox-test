# LayerStack Phase 1 Stage 00 Baseline

These two focused cases freeze the packaged legacy-v1 route before any Phase 1
candidate implementation exists.

## Contract

- `layerstack.phase1.baseline.legacy-route` uses only the public manager,
  runtime, and observability CLIs for sandbox and workspace lifecycle,
  write/publish/read/execute behavior, route evidence, and logical cleanup.
- `layerstack.phase1.baseline.restart-cleanup` enables the existing packaged
  one-shot publication failpoint, places its marker from outside the workload,
  and proves that a failure before staging does not change the active manifest
  or leave staging residue. The existing gateway replacement path then
  recovers the still-running sandbox and the retained workspace retries
  successfully.
- `/eos` is inspected only from the host with `docker exec`. No workload
  command can see or name `/eos`, and host inspection never substitutes for
  public correctness, route, or cleanup assertions.
- Every sandbox and workspace is registered immediately and released in LIFO
  order through the existing run-owned custody fixtures. No global cleanup is
  permitted.

Evidence is written beneath the existing bounded per-case artifact directory
as `layerstack-evidence-v1.json` and checked against
`e2e/schemas/layerstack_phase1/evidence-v1.schema.json`. The immutable
`corpus.json`, `tree.json`, and `root.json` fixtures under
`e2e/fixtures/layerstack_phase1/v1/` mirror the product-side v1 golden
contract; they are never regenerated by a test.

Stage 00 asserts only the legacy route and records raw measurements. It makes
no candidate, percentile, production-readiness, or later-stage claim.
